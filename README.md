## 这是什么？

一个 WorkBuddy skill，专为**临阵磨枪**设计。

你有没有过这种经历：考试就在眼前，教材还没翻完？这个 skill 不教你知识——它教你**怎么考**。

**核心哲学**：排除法、时间管理、猜测策略这些"元技能"，可以弥补 30-40% 的知识空白。

## 适用场景

- 🏃 还有 3 天就要考试，完全没准备
- 📚 学过但忘了，需要快速唤醒记忆
- 🎯 只求及格，不想花时间理解底层逻辑
- 🔄 考过但挂了，需要换一种策略重来

## 工作流程

```
输入考试名称 + 可用时间
        ↓
Phase 1: 信息收集（考试类型、目标分数、当前水平）
        ↓
Phase 2: 输出速成方案
  ├── 📅 速成计划（3天/7天/14天/30天模板）
  ├── 🎯 元技能包（排除法、关键词识别、概率猜测...）
  └── 📝 练习模块（模拟题、错题分析、进度跟踪）
        ↓
Phase 3: 渐进调整（根据反馈动态优化策略）
        ↓
Phase 4: 考前终极包（最后1天冲刺策略 + 考场应急预案）
```

## 支持的考试类型

| 类型 | 策略重心 |
|------|---------|
| 标准化选择题 | 排除法 + 关键词识别 + 概率猜测 |
| 简答/填空 | 关键词踩点 + 模板化答题 |
| 论述/作文 | 万能模板 + 结构优先 |
| 计算/解题 | 公式速记 + 题型识别 |
| 实操/机考 | 环境熟悉 + 标准操作演练 |
| 口试/面试 | 常见问题准备 + 结构化表达 |

## 设计原则

1. **不教知识，教"怎么考"** — 排除法可以帮你蒙对30%的题
2. **渐进式交互** — 先出框架，执行后反馈，动态调整
3. **承认边界** — 明确告知适用范围，不过度承诺

## 局限性

- 对**实操类考试**（编程机考、实验操作）效果有限
- 如果考试有**硬性知识门槛**，纯技巧可能无法通过
- 目标是**及格**而非高分——追求高分请系统学习

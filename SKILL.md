# Exam Cram — 考试速成框架

不教知识，教"怎么考"。核心哲学：**元技能优先**，在最短时间内通过考试。

## 设计原则

1. **不追求理解，追求得分** — 跳过底层逻辑，直接瞄准得分点
2. **元技能 > 知识量** — 排除法、时间管理、猜测策略可以弥补 30-40% 的知识空白
3. **渐进式交互** — 先出框架，用户执行后反馈，动态调整
4. **承认边界** — 明确告知适用范围，不过度承诺

## 触发条件

当用户表达以下意图时触发：
- "速成"、"突击"、"冲刺"、"临时抱佛脚"
- "快速通过XX考试"
- "还有X天就要考了"
- "怎么最快准备XX考试"
- "不求高分，只求及格"

## 工作流

### Phase 1: 信息收集

首次交互时，收集以下信息（缺什么问什么，不重复问已知的）：

| 信息 | 必要性 | 说明 |
|------|--------|------|
| 考试名称 | 必须 | 具体考试名称 |
| 可用时间 | 必须 | 距离考试还有多少天 |
| 目标分数 | 可选 | 默认"及格即可" |
| 考试形式 | 可选 | 选择题/问答/实操/混合（可尝试搜索） |
| 当前水平 | 可选 | 完全没学过/有一些基础/学过但忘了 |

**提问方式（自然对话，不要列清单）：**

> "哪个考试？还有几天？目标是及格还是高分？"

如果用户已经提供了部分信息，只问缺失的。如果信息足够，直接进入 Phase 2。

### Phase 2: 速成方案输出

基于收集的信息，输出以下三个模块：

#### A. 速成计划（时间表）

根据可用天数，选择对应模板（参见 `references/speed-plan-template.md`）：
- **3天以内**：极限冲刺方案
- **4-7天**：一周速成方案
- **8-14天**：两周速成方案
- **15-30天**：一月冲刺方案

输出格式：
```
## 📅 速成计划（X天方案）

### Day 1: [主题]
- [ ] 任务1 — [时长]
- [ ] 任务2 — [时长]
- ...

### Day 2: [主题]
...

### 核心原则
- [该方案的关键原则]
```

#### B. 元技能包

根据考试形式，从 `references/meta-skills.md` 选取适用的技巧组合。

按考试类型匹配（参见 `references/exam-patterns.md`）：
- 选择题为主 → 排除法 + 关键词识别 + 概率猜测
- 简答/填空 → 关键词踩点 + 模板化答题 + 分步得分
- 论述/作文 → 模板化写作 + 结构优先
- 计算/解题 → 公式速记 + 题型识别 + 分步得分
- 实操/机考 → 环境熟悉 + 标准操作演练
- 口试/面试 → 常见问题准备 + 结构化表达

输出格式：
```
## 🎯 元技能包

### 技能1: [技能名称]
**什么时候用：** [场景]
**怎么用：** [具体步骤]
**示例：** [用该考试的具体题目演示]

### 技能2: ...
```

#### C. 练习模块

根据考试信息，生成练习材料：

1. **模拟题生成**
   - 根据考试名称和题型，生成 10-20 道模拟题
   - 难度分布：60% 基础 + 30% 中等 + 10% 困难
   - 每题附带解题思路（不是答案解析，是"怎么猜"的思路）

2. **错题分析模式**
   - 用户提交错题 → AI 分析错误模式（知识盲区 vs 技术失误 vs 时间不足）
   - 输出针对性建议

3. **进度跟踪**
   - 用户每天汇报执行情况
   - AI 根据进展调整后续计划

输出格式：
```
## 📝 练习模块

### 模拟题（第1组）
**Q1.** [题目]
A. [选项A]
B. [选项B]
C. [选项C]
D. [选项D]

> 💡 速解思路：[用元技能解题的思路，不是知识讲解]

### 使用方法
1. 限时完成（X分钟）
2. 核对答案
3. 对错题：提交给我，我帮你分析错误模式
```

### Phase 3: 渐进调整

用户执行后，通过反馈动态调整：

| 用户反馈 | 调整策略 |
|---------|---------|
| "模拟题正确率低" | 分析错误模式，针对性补充 |
| "选择题没问题但问答题不会" | 切换技巧包，增加问答题模板 |
| "时间不够了" | 输出极限冲刺方案（参见 speed-plan-template.md） |
| "某个知识点反复错" | 生成该知识点的专项速记卡 |
| "模拟分数已经达到目标" | 切换为维持模式，减少练习量 |

### Phase 4: 考前终极包（考试前1天触发）

当用户说"明天就考了"或距离考试≤1天时，输出：

```
## 🚨 考前终极包

### 考前清单
- [ ] [考试相关的必带物品]
- [ ] 复习：只看错题笔记和速记卡
- [ ] 睡眠：至少7小时

### 最后冲刺
- 只看 [高频考点TOP5]
- 只看 [错题笔记]
- 不学新内容

### 考场应急预案
1. 遇到完全不会的题 → [具体策略]
2. 时间不够了 → [紧急抢分策略]
3. 心态崩了 → [10秒冷静法]
```

## 输出规范

1. **语言**：跟随用户语言（中文问中文答，英文问英文答）
2. **格式**：使用 Markdown，层次清晰，可直接阅读
3. **长度**：首次方案输出控制在 2000 字以内，避免信息过载
4. **语气**：直接、实用、不废话。像一个考过这个考试的朋友在给你支招
5. **边界声明**：首次输出时，用一句话说明适用边界

## 注意事项

1. **不生成完整学习资料** — 这是速成框架，不是学习框架
2. **不承诺分数** — 只提供策略，不保证结果
3. **鼓励反馈** — 主动询问执行效果，持续优化
4. **尊重用户节奏** — 如果用户说"来不及了"，不要批评，直接给应急方案

# Exam Cram — 考试速成框架

不教知识，教"怎么考"。核心哲学：**元技能 + 记忆术优先**，在最短时间内通过考试。

## 设计原则

1. **不追求理解，追求得分** — 跳过底层逻辑，直接瞄准得分点
2. **元技能 + 记忆术 > 知识量** — 排除法、时间管理、助记法可以弥补 30-40% 的知识空白
3. **自适应输出** — 根据用户可用时间动态调整输出深度，时间越少越精炼
4. **承认边界** — 明确告知适用范围，不过度承诺

## 触发条件

- "速成"、"突击"、"冲刺"、"临时抱佛脚"
- "快速通过XX考试"、"还有X天就要考了"
- "怎么最快准备XX考试"、"不求高分只求及格"

## 工作流

### Step 0: 恐慌模式（优先判断）

**如果用户明确表示时间极度紧张（≤1天）或处于焦虑状态，跳过常规流程，直接输出：**

1. 用 WebSearch 搜索该考试的关键信息（及格线、题型、分值分布）
2. 输出「🚨 应急速通包」（见下方模板）
3. 不问多余问题，不输出长篇方案

**应急速通包模板：**
```
## 🚨 [考试名] 应急速通包

### 你必须知道的3件事
1. 及格线/通过标准：[搜索结果]
2. 题型分布：[搜索结果]
3. 最容易拿分的部分：[搜索结果]

### 今晚做这3件事
1. [最紧急任务] — [时长]
2. [次紧急任务] — [时长]
3. [考前必看] — [时长]

### 明天考场上的保命策略
- [针对该考试题型的1-2个核心元技能]

### 考场应急
- 完全不会 → [策略]
- 时间不够 → [策略]
```

**否则，进入常规流程。**

---

### Step 1: 信息收集 + 考试情报搜索

**收集用户信息**（缺什么问什么，自然对话）：

| 信息 | 必要性 | 说明 |
|------|--------|------|
| 考试名称 | 必须 | 具体考试名称 |
| 可用时间 | 必须 | 距离考试还有多少天；每天能学几小时 |
| 目标分数 | 可选 | 默认"及格即可" |
| 考试形式 | 可选 | 选择题/问答/实操/混合 |
| 当前水平 | 可选 | 完全没学过/有一些基础/学过但忘了 |

**同时用 WebSearch 搜索考试情报**（根据考试名称搜索，取前3-5条有用信息）：

```
搜索关键词示例：
- "[考试名] 通过率 及格线"
- "[考试名] 题型 分值分布"
- "[考试名] 备考经验 速成"
- "[考试名] 2025/2026 大纲变化"
```

搜索结果用于：
- 补充/验证用户提供的考试形式
- 获取真实的及格线和通过率
- 了解常见备考陷阱
- 如果搜索失败，不阻塞流程，用已有信息继续

**提问方式（自然对话，不列清单）：**

> "哪个考试？还有几天？每天能学多久？目标是及格还是高分？"

信息足够后，根据考试类型匹配 `references/exam-patterns.md` 中的模式，进入 Step 2。

---

### Step 2: 速成方案输出

**根据可用时间自适应调整输出深度：**

| 可用时间 | 输出策略 | 详细程度 |
|---------|---------|---------|
| ≤3天 | 仅输出：应急计划 + 核心3个元技能 + 10道模拟题 | 极简 |
| 4-7天 | 输出：周计划 + 元技能包 + 练习模块 | 标准 |
| 8-14天 | 输出：完整计划 + 元技能包 + 练习模块 + 记忆术指导 | 详细 |
| 15-30天 | 输出：完整计划 + 全部模块 + 每周检查点 | 完整 |

#### A. 速成计划

根据可用天数和每日可用小时，从 `references/speed-plan-template.md` 选取模板并定制。

**关键定制规则：**
- 在职/每天≤2小时 → 使用「最小可行方案」模式（见 speed-plan-template.md）
- 全职/每天≥4小时 → 使用标准模板
- 优先标注每日「必做」和「选做」任务，让用户可以灵活裁剪

输出格式：
```
## 📅 速成计划（X天 | 每天~Y小时）

### Day 1: [主题]
- [ ] ★ 任务1 — [时长] [必做]
- [ ] 任务2 — [时长] [选做]

### 核心原则
- [2-3条该方案最关键的原则]
```

#### B. 元技能包

根据考试类型，从 `references/meta-skills.md` 选取 3-5 个最相关的技巧。

**选配规则：**
| 考试类型 | 必选技巧 |
|---------|---------|
| 选择题为主 | 排除法 + 关键词识别 + 概率猜测 |
| 简答/填空 | 关键词踩点 + 模板化答题 + 分步得分 |
| 论述/作文 | 模板化写作 + 结构优先 |
| 计算/解题 | 公式速记 + 题型识别 + 分步得分 |
| 实操/机考 | 环境熟悉 + 标准操作演练 |
| 口试/面试 | 结构化表达 + 常见问题准备 |

输出每个技巧时：**用该考试的具体题目演示**，而非泛泛说明。

#### C. 记忆加速模块（≥4天时输出）

从 `references/memory-techniques.md` 选取 2-3 个最适合的速记方法：
- 大量术语/定义 → 助记词链 + 间隔重复微循环
- 公式/流程 → 记忆宫殿 / 故事法
- 零散知识点 → 缩略词法 + 主动回忆卡

#### D. 练习模块

1. **模拟题** — 10-15道，60%基础+30%中等+10%困难，每题附"速解思路"（不是知识讲解）
2. **错题分析** — 用户提交错题后，分析错误模式：知识盲区 / 技术失误 / 时间不足
3. **进度追踪** — 建议用户每天简要反馈执行情况

---

### Step 3: 渐进调整

| 用户反馈 | 调整策略 |
|---------|---------|
| "模拟题正确率低" | 分析错误模式，针对性补充 |
| "选择题没问题但问答题不会" | 切换技巧包，增加问答题模板 |
| "时间不够了" | 切换为更短时间方案或恐慌模式 |
| "某个知识点反复错" | 生成该知识点的助记卡 |
| "记忆记不住" | 换一种记忆术方法 |
| "模拟分数已达标" | 切换为维持模式，减少练习量 |

---

### Step 4: 考前终极包（≤1天时触发）

```
## 🚨 考前终极包

### 考前清单
- [ ] 必带物品（根据考试类型列出）
- [ ] 只看速记卡和错题笔记
- [ ] 睡眠≥7小时

### 最后冲刺
- 高频考点 TOP 5
- 错题笔记回顾
- 不学新内容

### 考场应急预案
1. 完全不会 → [针对该考试的具体策略]
2. 时间不够 → [紧急抢分策略]
3. 心态崩了 → 10秒冷静法（放下笔→3次深呼吸→从最简单的题重新开始）
```

## 输出规范

1. **语言**：跟随用户语言
2. **格式**：Markdown，层次清晰
3. **长度**：首次输出 ≤1500字（恐慌模式 ≤800字），避免信息过载
4. **语气**：直接、实用、不废话。像一个考过这个考试的朋友在给你支招
5. **边界声明**：首次输出时用一句话说明适用边界

## 注意事项

1. **不生成完整学习资料** — 这是速成框架，不是学习框架
2. **不承诺分数** — 只提供策略，不保证结果
3. **WebSearch 优先补充真实情报** — 考试通过率、题型分布、大纲变化等
4. **鼓励反馈** — 主动询问执行效果，持续优化
5. **尊重用户节奏** — 如果用户说"来不及了"，直接给应急方案
